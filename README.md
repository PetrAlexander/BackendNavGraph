# Remote Graph DemoApp

Тестовое приложение для заметок на Android. Особенность приложения — навигация между экранами строится на основе графа, который загружается с удалённого сервера, а не задаётся статически в коде.

## Возможности

- Создание, редактирование и сохранение заметок
- Загрузка и кэширование графа навигации с сервера
- Построение навигации приложения на основе полученного графа

## Стек

- Kotlin
- Jetpack Compose — UI
- Retrofit — сетевые запросы
- Room — локальное хранилище
- Kotlin Coroutines / Flow

## Граф навигации

Приложение периодически запрашивает с сервера JSON-описание графа навигации и сохраняет его локально. Граф состоит из следующих частей:

- `version` — номер версии графа
- `startDestination` — стартовый экран
- `screens` — перечень экранов приложения, каждый описывается параметрами `route`, `presentation`, `variant`
- `transitions` — список переходов между экранами

Каждый переход состоит из:

- `from` — начальная точка (экран)
- `to` — конечная точка (экран)
- `event` — событие, инициирующее переход

### Пример графа

```json
{
  "version": 1,
  "startDestination": "notes",
  "screens": [
    { "route": "notes" },
    { "route": "editor", "presentation": "screen", "variant": "v1" }
  ],
  "transitions": [
    { "from": "notes", "event": "add_note", "to": "editor" },
    { "from": "notes", "event": "edit_note", "to": "editor" },
    { "from": "editor", "event": "save", "to": "notes" }
  ]
}
```

При получении нового графа приложение обновляет локальную копию и применяет изменения навигации без обновления приложения из стора.

## Запуск

1. Откройте проект в Android Studio.
2. Укажите адрес сервера в качестве параметра для baseUrl в Retrofit. На данный момент установлен - http://10.0.2.2:8080/
3. Запустите сборку на эмуляторе или устройстве.
